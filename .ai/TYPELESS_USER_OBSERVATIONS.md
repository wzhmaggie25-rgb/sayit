# Typeless 用户观察记录

> 日期：2026-06-26
> 证据类型：用户长期使用后的体感观察，尚待本机黑盒复现

## 注入失败后的界面行为

用户观察到：

1. Typeless 正常的录音/处理中小悬浮窗先结束工作；
2. 如果最终文字没有成功注入目标输入框，随后会弹出一个**尺寸明显更大的结果悬浮窗**；
3. 大悬浮窗中直接展示识别完成的文字，而不是只显示一条错误提示；
4. 窗口上有明确的**复制按钮**；
5. 用户可以在这个窗口中确认文字仍然存在，并主动复制；
6. 这种体验比仅提示“注入失败，请去历史页复制”更可靠、更有安全感。

## 当前可信程度

- 这是用户基于真实使用形成的明确体感，不是代码审计结论；
- 大悬浮结果窗与复制按钮可作为 Typeless 本机黑盒审计的最高优先级复现目标；
- 在复现前，不确定：
  - 大窗是否只在确认失败时出现；
  - 是否也用于无法验证的注入；
  - 文字是否已经自动放入系统剪贴板；
  - 复制按钮是否只是再次复制；
  - 是否有重新注入/重试按钮；
  - 窗口是否自动关闭；
  - 点击窗口是否抢占原应用焦点；
  - 关闭窗口后是否仍能从历史找回。

## 审计时必须优先验证

### 触发条件

分别制造：

1. 原输入窗口在识别完成前关闭；
2. 原窗口存在但焦点切换到另一个窗口；
3. 目标控件变成不可编辑；
4. 在没有输入框的桌面/普通窗口开始或结束录音；
5. 可编辑窗口存在，但阻止或忽略模拟输入；
6. 长文本注入失败。

记录哪一种情况会触发大结果悬浮窗。

### 大结果悬浮窗

记录：

- 窗口大致尺寸和位置；
- 是否显示全文、截断文本或可滚动文本；
- 是否可以选中文字；
- 是否有复制按钮；
- 按钮的确切文案和图标；
- 是否有关闭、重试、历史等其他按钮；
- 点击复制后的视觉反馈；
- 点击复制后系统剪贴板是否变为结果文字；
- 窗口是否抢焦点；
- 用户按 Ctrl+V 是否可直接粘贴；
- 是否自动关闭以及持续时长；
- 手动关闭后识别结果是否仍在历史中。

### 剪贴板

测试前设置无隐私哨兵文本。分别记录：

- 大窗出现前，剪贴板是否已经变成识别结果；
- 不点击复制按钮时，剪贴板是否变化；
- 点击复制按钮后，剪贴板何时变化；
- 关闭大窗后，剪贴板是否保留结果；
- 成功注入时是否不会出现大窗，并恢复原剪贴板。

## 对 SayIt 的暂定产品启示

在审计结果出来前，SayIt 可以先确定以下方向，但不要声称是 Typeless 的确切实现：

1. 注入失败不能等同于识别失败；
2. 识别结果必须直接展示，不能要求用户去历史页寻找；
3. 失败时应出现比日常录音浮窗更明显的结果卡片；
4. 结果卡片至少包含：
   - 完整文字或可滚动文字；
   - 复制按钮；
   - 简短说明“未能自动输入”；
   - 关闭按钮；
5. 复制成功后给出明确反馈，例如“已复制”；
6. 结果窗口不应阻塞、强抢焦点或让用户丢失原工作位置；
7. 历史仍保留该条结果；
8. 是否自动预先复制到剪贴板，等待黑盒审计后再决定。

## 推荐的 SayIt 暂定文案

标题：

```text
未能自动输入
```

说明：

```text
文字已生成，你可以复制后粘贴到需要的位置。
```

按钮：

```text
复制文字
```

复制后：

```text
已复制
```

以上文案只是 SayIt 产品草案，不代表 Typeless 原文。
