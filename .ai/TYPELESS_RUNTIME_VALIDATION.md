# Typeless 运行时人工验证结果

> 日期：2026-06-27
> 证据类型：**用户现场人工黑盒验证**
> 状态：**RUNTIME_VALIDATED_FOR_NO_EDITABLE_TARGET**

## 重要结论

本次已经验证的不是笼统的“Typeless 注入失败”，而是更准确的场景：

> **当结束语音输入时，当前光标不在可编辑输入框内，Typeless 不会覆盖原剪贴板，也不会把文字强行注入其他窗口，而是弹出一个较大的结果悬浮窗，让用户查看并主动复制结果。**

这与此前“失败后立即自动把 final_text 放进剪贴板”的假设不同。

---

## 一、触发条件

`[用户黑盒实测]`

当光标离开输入框，即当前焦点不在可编辑输入控件内时：

- Typeless 不把识别结果强行注入当前非输入区域；
- 正常的小录音/处理浮窗结束；
- 随后弹出一个明显更大的结果悬浮窗。

目前已经验证的是“无有效可编辑目标”场景。

尚未验证：

- 原输入框仍然存在，但 Ctrl+V/SendInput 被目标程序拒绝时是否显示同一窗口；
- 仅部分文字注入成功时如何处理；
- 应用权限差异导致的真实注入失败是否走同一路径。

因此产品和代码中应将该场景命名为：

```text
no_editable_target
```

或：

```text
target_not_editable
```

不要仅笼统归类为 `recognition_failed`。

---

## 二、大结果悬浮窗内容

`[用户黑盒实测]`

窗口包含：

1. 第一行显示“最后转录的文字”；
2. 第二行显示刚才语音输入所对应的完整转述文字；
3. 当第二行内容较长时，末尾显示省略号；
4. 文字下方有一个“复制”按钮；
5. 右上角有关闭按钮。

说明：

- 当前观察确认窗口至少展示两层信息；
- 第一行与第二行的准确产品字段名称尚未从 UI 文案或代码语义中确认；
- 不要擅自改写为“标题 + 正文”或“原文 + AI 润色”，实现前应以用户实际看到的界面为准；
- 长内容目前观察到省略号，尚未确认是否支持滚动、展开或悬停查看全文。

## 三、剪贴板行为

`[用户黑盒实测]`

测试前剪贴板内容：

```text
TYPELESS_SENTINEL_20260627
```

### 不点击复制按钮

- 大结果窗出现后，系统剪贴板仍保持：

```text
TYPELESS_SENTINEL_20260627
```

- 此时按 `Ctrl+V`，粘贴的是原来的哨兵文字；
- Typeless 没有自动把本次语音结果写入剪贴板。

### 点击复制按钮

- 点击后，本次语音输入结果才写入系统剪贴板；
- 随后按 `Ctrl+V`，可以粘贴刚才的语音输入文字；
- 复制按钮左侧出现绿色勾选反馈；
- 绿色勾出现后，整个结果窗口随即消失。

由此可以确认：

> **Typeless 在无有效输入目标时，优先保护用户原剪贴板；只有用户主动点击复制，才用语音结果替换剪贴板。**

---

## 四、已经确认的交互流程

`[用户黑盒实测]`

```text
开始语音输入
→ 当前焦点不在可编辑输入框
→ 小录音/处理浮窗完成
→ 弹出大结果悬浮窗
→ 展示最近转录信息和语音结果摘要
→ 原剪贴板保持不变
→ 用户点击“复制”
→ 按钮左侧显示绿色勾
→ 语音结果写入剪贴板
→ 大结果窗口消失
```

右上角关闭按钮可用于放弃本次复制操作。

尚未明确：

- 点击关闭后是否仍可从历史找回；
- 窗口是否存在自动消失超时；
- 长文字能否展开或滚动；
- 复制后消失是否立即发生或有短暂动画；
- 窗口是否抢焦点。

## 五、对 SayIt 产品策略的直接修订

### 1. 无有效输入框时，不应自动覆盖剪贴板

旧任务中的以下要求应被本验证结果覆盖：

```text
无输入目标或不可确认时，立即把 final_text 放入剪贴板
```

新的默认策略：

```text
无有效可编辑目标
→ 保留原剪贴板
→ 显示大结果卡片
→ 用户主动点击复制后才替换剪贴板
```

### 2. 不应显示“识别失败”

此时语音识别已经成功，问题只是当前没有可接收文字的输入框。

建议状态：

```text
recognition_success + no_editable_target + result_card_shown
```

不得归类为：

```text
recognition_failed
```

### 3. SayIt 需要新增独立的大结果卡片

结果卡片至少包含：

- 最近转录信息；
- 本次最终语音结果的预览；
- 长文本省略或安全滚动；
- 复制按钮；
- 关闭按钮；
- 复制成功绿色勾反馈；
- 复制成功后自动关闭。

### 4. 复制动作必须由用户主动触发

在 `no_editable_target` 场景：

- 显示卡片时不修改剪贴板；
- 点击复制后再写入 `final_text`；
- 复制成功后显示绿色勾；
- 随后关闭卡片；
- 原剪贴板只有在用户明确点击复制后才被替换。

### 5. 与真正注入失败分开处理

建议至少区分：

```text
verified_success
no_editable_target
injection_failed_with_valid_target
recognition_failed
```

本次 Typeless 运行时验证只明确支持 `no_editable_target` 的大结果卡片行为。

对于 `injection_failed_with_valid_target` 是否也使用同一卡片，仍需单独测试或由 SayIt 自己做产品决策。

## 六、SayIt 建议实现的第一版

### 触发条件

当以下条件成立：

- ASR/AI 已得到非空 `final_text`；
- 当前没有有效、可编辑、可恢复的输入控件；

则：

1. 不调用自动 clipboard fallback；
2. 不显示“识别失败”；
3. 保存历史，状态为 `no_editable_target`；
4. 打开大结果卡片；
5. 卡片展示 final_text 预览；
6. 用户点击复制后写入剪贴板；
7. 显示绿色勾；
8. 关闭结果卡片。

### 推荐文案草案

第一行：

```text
最后转录的文字
```

第二行：

```text
{final_text 预览}
```

按钮：

```text
复制
```

复制后：

```text
✓ 已复制
```

文案草案不代表 Typeless 的逐字原文；视觉和措辞可按 SayIt 品牌调整。
