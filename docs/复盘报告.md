## 4. API 配置诊断

### 4.1 诊断脚本

已创建 `docs/diag_test.py`，运行结果摘要：

```
DashScope API Key:  (空)          → 主 ASR 引擎不可用
Volcengine ASR Key: (空)          → 火山 ASR 不可用
Volcengine AI Key:  (空)          → 豆包 AI 不可用
DeepSeek Key:       (空)          → DeepSeek 不可用
ai_providers: [aliyun(enabled,key空), volcengine(enabled,key空)]
所有 Python 依赖:   已安装 ✓
ONNX model.onnx:   缺失
ONNX tokens.json:  缺失  
SenseVoice model.pt: 存在 ✓
```

### 4.2 根本原因

1. **config.json 写入安全机制过于激进**: `config_store.py:126-158` 的 `save(_strip_keys=True)` 在 server.py 的 `set_config()` 中被调用，所有 API key 在保存时会被清空。前端 settings.html 通过 `saveConfig()` 写入 key 后，Key 先被临时写入内存，但 config.json 持久化时被清空。

2. **Key 传递链断裂**:
   - 前端 `saveConfig()` (settings.html:845) → 调用 `window.sayit.setConfig(cfg)` → IPC invoke → `main.js:167` → `api('POST', '/api/config', data)` → `server.py:89` → `config.set()` → `config.save(_strip_keys=True)` → Key 被写入空字符串
   - 下次启动时，从 config.json 加载 → Key 为空 → ASR/AI 全部不可用

3. **settings.html 的 localStorage 缓存在充数**: settings 页把 API Key 也存了 localStorage 作为"缓存"，但这让 Key 存于明文本地存储中，有安全隐患。

4. **Key 的双源不一致**: 设置页中火山方舟用"arkKey"字段名，但后端 config.json 实际结构是 `volcengine.asr.api_key`，两者映射在 `saveConfig()` 中有 `cfg.volcengine.asr.access_token = arkVal` 的逻辑。

### 4.3 修复方向（仅供后续参考，本次不改代码）

- 将 config.json 的 API Key 存储改用独立文件（如 `.env` 或 keyring）
- 修复 `set_config()` 中的 `save(_strip_keys=True)` 逻辑——在保存前先保留内存中的真实 key，仅磁盘存储用 _strip_keys
- 或者改方案：API Key 不通过 config.json 传递，改为从 settings 页面直接存入独立配置文件
