# Sayit 调试夹具说明

CC 闭环诊断基础设施。无需用户录音，curl 即可驱动完整注入/ASR/AI 链路。

---

## 前置条件

1. 后端已启动：`cd d:\Soft\code\sayiy1.1\sayit_cg && python server.py`
2. 确认新代码在跑：启动日志含 `[AI-KEY]` tag
3. 确认端口未被占：无 `[Errno 10048]`（有则走下方 Hermes 排查 SOP）

## 调试端点

### `POST /api/debug/inject` — 注入隔离测试

绕过录音/ASR/AI，直接注入固定文本。

```bash
# 基础用法
curl -s -X POST http://127.0.0.1:17890/api/debug/inject \
  -H "Content-Type: application/json" \
  -d '{"text":"测试"}'

# 返回示例
{"ok":true,"text_len":2,"target_proc":"notepad.exe","target_class":"Notepad","target_title":"*新建 文本文档.txt - Notepad"}

# paste_only 模式：跳过 _release_modifiers + UIA，直接 Ctrl+V
curl -s -X POST http://127.0.0.1:17890/api/debug/inject \
  -H "Content-Type: application/json" \
  -d '{"text":"测试","paste_only":true}'
```

用途：验证注入链路正确性，隔离 preamble（`_release_modifiers`/UIA）副作用。

### `POST /api/debug/pipeline-text` — 文本全链路

文本 → corrector（AI 纠错）→ injector。

```bash
curl -s -X POST http://127.0.0.1:17890/api/debug/pipeline-text \
  -H "Content-Type: application/json" \
  -d '{"text":"你好世界"}'

# 返回示例
{"ok":true,"original":"你好世界","corrected":"你好，世界。","target_proc":"notepad.exe"}
```

用途：验证 AI 纠错 + 注入全链路（跳过录音/ASR）。

### `POST /api/debug/release-only` — _release_modifiers 隔离

只发 modifier keyup，不注入文本。用于排查注入 preamble 副作用。

```bash
curl -s -X POST http://127.0.0.1:17890/api/debug/release-only

# 返回示例
{"ok":true,"note":"release_modifiers sent, check Notepad for stray chars"}
```

用途：验证 `_release_modifiers()` 是否产生多余字符（已通过 GetAsyncKeyState 守卫修复）。

### `GET /api/debug/wav-check` — WAV 文件信息

返回最近一次录音的 WAV 采样率、时长、大小。

```bash
curl -s http://127.0.0.1:17890/api/debug/wav-check

# 返回示例
{"ok":true,"path":"C:\\Users\\46136/Desktop/sayit_last.wav",
 "framerate":16000,"nframes":111616,"channels":1,"sampwidth":2,
 "duration_s":6.976,"size_bytes":223276}
```

用途：验证 WAV 采样率链路、音频数据完整性。

## 日志路径

所有诊断日志写入：`%APPDATA%/Sayit/sayit.log`

### 诊断 tag 速查

| Tag | 含义 | 何时出现 |
|-----|------|---------|
| `[WAV-CHECK]` | WAV 头 vs PCM 数据一致性 | ASR 引擎写 WAV 时 |
| `[AI-KEY]` | AI provider key 来源/endpoint | 后端启动时 |
| `[AUDIO-DEVICE]` | 设备原生采样率 vs 请求值 | 录音启动时 |
| `[INJECT-PATH]` | 注入路径（UIA/Clipboard/SendInput） | 每次注入 |
| `[INJECT-PRE]` | 注入前文本/修饰键状态 | 每次注入 |
| `[INJECT-POST]` | 注入结果（via=UIA/Clipboard/SendInput） | 每次注入 |
| `[VOLC-ASR-EP]` | 火山 ASR endpoint scheme/host | 火山 ASR 调用前 |
| `[PORT-BUSY]` | 17890 端口占用详情 | 启动时端口被占 |
| `[LLM-REQ]` / `[LLM-RESP]` | AI LLM 请求/响应 | AI 纠错调用时 |

### 常用 grep

```bash
# 注入全流程
grep -E "\[INJECT-PATH\]|\[INJECT-PRE\]|\[INJECT-POST\]" "$APPDATA/Sayit/sayit.log" | tail -20

# 注入乱码排查（看 SendInput 是否有 ascii_hits）
grep "route=sendinput" "$APPDATA/Sayit/sayit.log"

# AI 调用链路
grep -E "\[AI-KEY\]|\[LLM-REQ\]|\[LLM-RESP\]" "$APPDATA/Sayit/sayit.log"

# 火山 ASR 端点
grep "\[VOLC-ASR-EP\]" "$APPDATA/Sayit/sayit.log"
```

## Hermes Agent 占端口排查 SOP

**症状**：`python server.py` 报 `[Errno 10048]`，手动 `taskkill` 后几秒端口又被占。

**根因**：Hermes Agent（Claude Code 基础设施）自动托管并重启 Sayit 后端。

**处置流程**：

```powershell
# 1. 查谁在占 17890
netstat -ano | findstr :17890
# 输出示例: TCP  127.0.0.1:17890  0.0.0.0:0  LISTENING  30756

# 2. 看进程名/路径
tasklist /FI "PID eq 30756"
# 若是 python.exe，继续看完整路径
powershell -Command "Get-Process -Id 30756 | Select-Object Id, ProcessName, Path"

# 3. 查父进程链
powershell -Command "(Get-WmiObject Win32_Process -Filter 'ProcessId=30756').ParentProcessId"
# 若父进程是 hermes-agent/venv/python.exe → 即 Hermes Agent 在托管

# 4. 全杀（开发期间不宜共存）
powershell -Command "Get-Process -Name 'python*' -ErrorAction SilentlyContinue | Stop-Process -Force"

# 5. 等 5 秒确认未自动重启
Start-Sleep 5
Get-Process -Name "python*"   # 应空
netstat -ano | findstr :17890  # 应空

# 6. 手动起新代码
cd d:\Soft\code\sayiy1.1\sayit_cg
python server.py
# 盯前 3 秒：无 10048、有 [AI-KEY] = 新代码单实例在跑
```

### 如果反复被占

检查 Hermes Agent 配置（`%LOCALAPPDATA%\hermes\hermes-agent\`）是否有 server.py 托管规则，或直接退出托盘中的 Hermes Agent。

## 采样率「一耳朵」验证

录一句后，桌面会出现 `sayit_last.wav`：

| 听感 | 诊断 |
|------|------|
| 正常清晰 | 采样率链路 OK |
| 快/尖 | header rate < 实际 rate |
| 慢/闷 | header rate > 实际 rate |
| 噪 | 位深/字节序错 |
| 静 | 默认设备非麦克风 |

验证完可删除桌面临时文件。
